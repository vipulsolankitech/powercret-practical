Task Details --- # Task 1: Custom Post Type – Testimonials ### Overview A custom plugin named **"Testimonial"** was created to manage testimonials. ### Implementation Details * Created a custom plugin with a proper plugin header. * Added an `ABSPATH` check to prevent direct access to plugin files. * Registered the `testimonial` custom post type using `register_post_type()` and the `init` action. * Created a separate `custom_fields.php` file and included it in the main plugin file. * Added three custom fields using `add_meta_box()` and the `add_meta_boxes` action. * Saved custom field values using the `save_post` action. * Added custom columns to the testimonial listing page using the `manage_testimonial_posts_custom_column` action. * Enabled column sorting using: * `manage_edit-testimonial_sortable_columns` * `pre_get_posts` * Created a shortcode using `add_shortcode()` to display testimonials. * Retrieved testimonial data using `WP_Query` with a configurable limit. * Created the page: ``` /testimonial-front ``` and used the shortcode to display the testimonials. --- # Task 2: WooCommerce Gift Wrap Add-on ### Overview A custom plugin named **"Gift Wrap for WooCommerce"** was developed to provide an optional gift wrapping feature. ### Implementation Details * Installed and configured WooCommerce with Cash on Delivery and demo products. * Created a custom plugin with a proper plugin header. * Added an `ABSPATH` check to prevent direct access. * Displayed a Gift Wrap checkbox on the cart page using: ```php woocommerce_after_cart_table ``` * Added the gift wrap charge as a fee using: ```php add_fee() ``` This ensures that the charge is carried through checkout, stored in the order, and displayed in the admin area. * Enqueued assets using `wp_enqueue_scripts`. * Passed AJAX variables such as: * Nonce * `admin-ajax.php` URL * Added a change event for the checkbox and sent the following parameters via AJAX: * action * gift_wrap * nonce * Handled AJAX requests using: ```php wp_ajax_* ``` * Managed the Gift Wrap selection through WooCommerce sessions. * Recalculated cart totals and refreshed totals using: ```php woocommerce_cart_totals() ``` * Used the following action to reset the gift wrap option when the cart becomes empty: ```php woocommerce_cart_emptied ``` ### Result The additional Gift Wrap charge is visible: * Cart page * Checkout page * Order summary * WooCommerce admin order screen --- # Task 3: Security Improvements ### Overview The provided vulnerable code was reviewed and secured. ### Security Enhancements * Added `isset()` checks before accessing `user_id`. * Sanitized user input to prevent invalid or malicious data. * Used prepared statements to protect against SQL injection. * Escaped output properly to prevent unwanted script execution. * Considered implementing capability checks, but since the original purpose of the function was unclear, no permission restrictions were added. * Debugged the code during page load using the `init` action. * Rewrote the improved code in: ```text wp-content/plugins/Testimonial/code_debug.php ``` --- # Task 4: AI Integration – Post Summarizer ### Overview A plugin named **"Blog Summary"** was created to generate summaries for default WordPress posts. ### Implementation Details * Added an `ABSPATH` check to prevent direct access. * Enqueued assets using `wp_enqueue_scripts`. * Localized AJAX variables such as: * Nonce * `admin-ajax.php` URL * Created the function: ```php blog_button_after_content() ``` which renders: * The "Summarize Blog" button * A container to display the response This functionality is injected after the post content using the `the_content` filter. * Created: ```text get_summary_result.php ``` to handle AJAX requests. * Triggered AJAX requests when the user clicks the **"Summarize Blog"** button. * Created: ```php get_blog_summary() ``` and registered it using: ```php wp_ajax_* ``` * Implemented a dummy response containing the dynamic post title. * Displayed the response below the button. * Added a sample implementation using `wp_remote_post()` for real API integration and kept it commented because no API key was available. * Added a simple loading indicator showing: ```text Loading... ``` while the request is being processed. * Stored the API key in a configuration file instead of exposing it directly inside plugin files. --- **Developed by:** **Vipul Solanki**
